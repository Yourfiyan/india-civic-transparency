# India Civic Transparency Platform — AI Instructions

> **READ THIS FIRST.** This file tells you everything about this project so you
> can get it running and start working immediately.

## What This Project Is

A full-stack India civic data dashboard: interactive Leaflet map with district
boundaries, crime registration heatmap, infrastructure project markers, Supreme
Court case browser with pagination, and an analytics dashboard. Dark theme UI
with Tailwind CSS.

## Environment: Google Colab + VS Code Tunnel

This project runs in **Google Colab** — NOT a normal dev machine. Key facts:

- Workspace lives on **Google Drive** at:
  `/content/drive/MyDrive/Colab_VSCode/workspace/india-civic-transparency/`
- **PostgreSQL data is EPHEMERAL** — lives on the Colab VM, wiped every restart.
- **esbuild binary** can't execute from Google Drive FUSE — must be copied to `/tmp/esbuild-bin`.
- **WebGL is disabled** in Colab's iframe browser — that's why we use Leaflet Canvas 2D, not MapLibre.
- **NEVER** run `kill $(lsof -t -i:PORT)` blindly — it can kill the VS Code tunnel process.

## First Thing To Do Every Session

Run the bootstrap script. It installs PostgreSQL, creates the DB, seeds all data,
fixes esbuild, and starts both servers:

```bash
cd /content/drive/MyDrive/Colab_VSCode/workspace/india-civic-transparency
bash bootstrap.sh
```

This takes ~60 seconds and gets everything running. After it completes:
- Backend API: http://localhost:3000
- Frontend: http://localhost:5173
- Health check: http://localhost:3000/api/health

## Map Layers (MapView.tsx)

1. **District Boundaries** — GeoJSON from TopoJSON (quantization 1e7), colored by
   composite score (red→yellow→green). Click for detail, hover for tooltip.
2. **Crime Registrations** — Circle markers from `/api/crime/geo`. Size proportional
   to case count. Red for data, gray for districts with no crime data. Uses LEFT JOIN
   so all 20 districts appear.
3. **Infrastructure** — Circle markers from `/api/infrastructure/geo`. Color by status:
   green (completed), yellow (in_progress), orange (sanctioned). Sample dataset (30 projects).

## Key Gotchas

1. **esbuild EACCES**: Google Drive FUSE doesn't support execute permission.
   Fix: `cp frontend/node_modules/@esbuild/linux-x64/bin/esbuild /tmp/esbuild-bin`
   Then: `ESBUILD_BINARY_PATH=/tmp/esbuild-bin npx vite ...`

2. **Port conflicts**: Check before killing: `ps aux | grep "node server" | grep -v grep`
   Kill only the specific PID, never `lsof -t -i:PORT`.

3. **StrictMode double-mount**: MapView uses a `cancelled` flag in useEffect to prevent
   race conditions from React 18 StrictMode double-init.

4. **TopoJSON quantization**: Set to 1e7 in `generate-topojson.js`. Lower values
   (like 1e5) crush polygon detail into squares.

5. **Crime data coverage**: Only 11 of 20 districts have crime_stats rows. The
   `/api/crime/geo` endpoint uses LEFT JOIN + COALESCE so all 20 appear on the map.

If bootstrap already ran and you just need to restart:

```bash
# Backend
cd backend
PGUSER=civic PGPASSWORD=civic PGDATABASE=civic_transparency PGHOST=localhost \
  node server.js &

# Frontend
cd frontend
ESBUILD_BINARY_PATH=/tmp/esbuild-bin npx vite --port 5173 --host 0.0.0.0 &
```
